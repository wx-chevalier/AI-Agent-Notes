## 一、架构总览

### 1. 整体架构设计

基于 DeepAgents 构建 Code Agent 的核心是**「框架调度层 + 代码能力层 + 工具链层」**的三层架构：

- **框架调度层**：由 DeepAgents 提供，负责任务分解（`write_todos`）、子代理管理（`task` 工具）、状态跟踪和上下文维护。
- **代码能力层**：集成代码专用 LLM（如 Kimi、GPT-4o Code），提供代码生成、解释、调试等核心能力。
- **工具链层**：通过 DeepAgents 的工具机制扩展代码相关工具（文件操作、语法检查、测试执行等）。

架构示意图：

```
用户需求 → DeepAgents（任务分解/调度）→ 子代理（专项编码任务）
                                          ↓
                      工具链（文件操作/代码执行/测试）←→ 代码LLM（生成/调试）
```

### 2. 关键组件配置

#### （1）基础 Agent 初始化

通过 `create_deep_agent` 函数配置核心参数，重点包括模型、工具和子代理：

```python
from deepagents import create_deep_agent
from langchain_community.chat_models import KimiChat  # 示例：Kimi模型

# 初始化代码专用LLM
code_llm = KimiChat(
    api_key=os.environ["KIMI_API_KEY"],
    model_name="kimi-code-pro",  # 假设的代码优化模型
    temperature=0.1  # 低温度保证代码稳定性
)

# 定义代码相关工具（基础工具+自定义工具）
code_tools = [
    # DeepAgents内置文件工具
    "ls", "read_file", "write_file", "edit_file", "glob", "grep", "execute",
    # 自定义代码工具（示例）
    code_lint_tool,  # 代码 lint 检查
    unit_test_generator  # 单元测试生成
]

# 创建Code Agent
code_agent = create_deep_agent(
    model=code_llm,
    tools=code_tools,
    system_prompt="你是专业Code Agent，擅长Python/Java代码开发，遵循PEP8规范，生成可运行代码并自动测试。",
    subagents=code_subagents  # 子代理配置（见下文）
)
```

#### （2）子代理设计

针对代码开发的细分场景（如前端、后端、测试），配置专业化子代理：

```python
# 测试子代理示例
test_subagent = {
    "name": "test-generator",
    "description": "生成单元测试代码，覆盖边界条件",
    "system_prompt": "使用pytest为给定代码生成测试用例，确保覆盖率>80%",
    "tools": ["read_file", "write_file", "execute"],  # 读取源码、写入测试、执行测试
    "model": code_llm  # 复用代码LLM
}

# 代码重构子代理示例
refactor_subagent = {
    "name": "code-refactor",
    "description": "优化代码结构，提升性能和可读性",
    "system_prompt": "遵循SOLID原则重构代码，保留功能不变，添加详细注释",
    "tools": ["read_file", "edit_file", "code_lint_tool"]
}

code_subagents = [test_subagent, refactor_subagent]
```

#### （3）工具链扩展

除 DeepAgents 内置的文件和执行工具外，需补充代码专用工具：

- **代码检查工具**：集成 `pylint`/`flake8` 进行语法校验（通过 `execute` 调用）。
- **依赖管理工具**：自动解析 `requirements.txt` 或 `pyproject.toml`，生成安装命令。
- **测试执行工具**：调用 `pytest`/`unittest` 执行测试并返回结果。
- **版本控制工具**：封装 `git` 命令（`commit`/`diff`），支持代码提交跟踪。

#### （4）长期记忆配置

利用 DeepAgents 的 `StoreBackend` 保存项目上下文，确保跨会话一致性：

```python
from deepagents.backends import CompositeBackend, StoreBackend
from langgraph.store.memory import InMemoryStore

# 持久化存储代码风格、项目结构等信息
memory_store = InMemoryStore()  # 生产环境可替换为Redis等

code_agent = create_deep_agent(
    model=code_llm,
    tools=code_tools,
    backend=CompositeBackend(
        default=StateBackend(),  # 临时状态
        routes={"/code_context/": StoreBackend(store=memory_store)}  # 持久化代码上下文
    )
)
```

## 二、关键注意点

### 1. 模型适配与优化

- **代码专用模型优先**：选择对代码生成友好的模型（如 Kimi Code、GPT-4o Code），避免通用模型在语法细节上的缺陷。
- **参数调优**：设置较低的 `temperature`（0.1-0.3）保证代码逻辑稳定，`max_tokens` 需覆盖长代码块（建议 8192+）。
- **提示词工程**：在 `system_prompt` 中明确代码规范（如命名风格、注释要求、框架版本），示例：
  ```python
  system_prompt = "生成Python代码需遵循PEP8，函数必须包含类型注解和文档字符串，优先使用Python 3.10+特性。"
  ```

### 2. 工具安全性与沙箱隔离

- **执行环境隔离**：通过 DeepAgents 的 `SandboxBackendProtocol` 配置沙箱（如 Docker、Modal），限制 `execute` 工具的系统权限，避免恶意代码执行。

  ```python
  from deepagents.backends.sandbox import ModalSandboxBackend  # 示例沙箱

  code_agent = create_deep_agent(
      model=code_llm,
      backend=ModalSandboxBackend()  # 所有命令在Modal沙箱中执行
  )
  ```

- **工具权限控制**：对 `edit_file`、`execute` 等危险工具启用「人工确认」（通过 `interrupt_on` 参数）：
  ```python
  code_agent = create_deep_agent(
      interrupt_on={"execute": True, "edit_file": True}  # 执行/修改文件需用户确认
  )
  ```

### 3. 任务分解与上下文管理

- **细粒度任务拆分**：利用 `write_todos` 工具将复杂任务（如「开发用户系统」）拆分为可执行步骤：
  ```
  1. 设计数据模型（User表结构）
  2. 实现注册接口（含密码加密）
  3. 编写单元测试
  4. 生成API文档
  ```
- **上下文窗口管理**：通过 DeepAgents 的 `SummarizationMiddleware` 自动压缩历史对话，避免代码上下文溢出：

  ```python
  # 配置摘要中间件（当上下文达85% tokens时自动压缩）
  from deepagents.middleware import SummarizationMiddleware

  code_agent = create_deep_agent(
      middleware=[SummarizationMiddleware(trigger=("fraction", 0.85), keep=("fraction", 0.1))]
  )
  ```

### 4. 代码质量保障

- **自动化测试闭环**：通过子代理实现「生成代码 → 执行测试 → 修复错误」的闭环：
  1. 主代理生成核心代码。
  2. 调用 `test-generator` 子代理生成测试用例。
  3. 执行测试并捕获错误，由主代理调用 LLM 修复。
- **版本控制集成**：通过 `git` 工具跟踪代码变更，支持回滚和历史对比，示例工具调用：
  ```json
  {
    "name": "execute",
    "args": { "command": "git diff > changes.patch" }
  }
  ```

### 5. 扩展性与定制化

- **项目特定配置**：通过 `.deepagents/agent.md` 注入项目专属规则（如框架版本、依赖限制），DeepAgents CLI 会自动加载项目级配置。
- **中间件扩展**：开发自定义中间件处理代码特定逻辑（如自动导入优化、依赖冲突检查）：
  ```python
  class DependencyCheckMiddleware(AgentMiddleware):
      def pre_process(self, state: AgentState) -> AgentState:
          # 检查代码中的未声明依赖
          code = extract_code_from_state(state)
          missing_deps = detect_missing_dependencies(code)
          if missing_deps:
              state["messages"].append(HumanMessage(content=f"需安装依赖：{missing_deps}"))
          return state
  ```

## 三、典型流程示例（代码生成场景）

1. **用户输入**：「用 FastAPI 实现一个获取用户列表的接口，包含分页功能」。
2. **任务分解**：DeepAgents 调用 `write_todos` 生成步骤：
   - 定义 User 数据模型（Pydantic）。
   - 实现分页查询接口（含 skip/limit 参数）。
   - 编写接口测试用例。
3. **代码生成**：主代理调用 LLM 生成模型代码，通过 `write_file` 写入 `models/user.py`。
4. **子代理协作**：调用 `test-generator` 子代理生成测试代码，保存至 `tests/test_api.py`。
5. **执行与调试**：调用 `execute` 工具运行 `pytest`，若失败则由主代理分析错误并修复。
6. **结果返回**：整理代码文件和测试报告，反馈给用户。

通过上述框架和注意点，基于 DeepAgents 的 Code Agent 可实现从需求到可运行代码的全流程自动化，同时保证代码质量和安全性。

## 四、代码实现

### 4.1 Code Agent 初始化

```python
import os
import uuid
from typing import List, Dict, Any
from deepagents import create_deep_agent
from deepagents.backends import CompositeBackend, StoreBackend, ModalSandboxBackend
from deepagents.middleware import SummarizationMiddleware
from deepagents.tools import Tool
from langchain_community.chat_models import KimiChat
from langchain_core.messages import HumanMessage, AIMessage
from langgraph.store.memory import InMemoryStore

# --------------------------
# 1. 初始化代码专用 LLM（Kimi Code）
# --------------------------
def init_kimi_code_llm() -> KimiChat:
    """初始化 Kimi Code 模型，配置代码生成参数"""
    return KimiChat(
        api_key=os.environ["KIMI_API_KEY"],
        model_name="kimi-code-pro",  # Kimi 代码优化模型
        temperature=0.2,  # 低温度保证代码稳定性
        max_tokens=8192,  # 支持长代码块生成
        streaming=True
    )

# --------------------------
# 2. 定义代码专用工具（基础工具 + 自定义工具）
# --------------------------
def get_code_tools() -> List[Tool]:
    """获取代码处理所需工具集"""
    # 基础文件操作工具（DeepAgents 内置）
    base_tools = [
        "ls", "read_file", "write_file", "edit_file", "glob", "grep", "execute"
    ]

    # 自定义代码工具：代码检查（调用 pylint）
    def code_lint_tool(file_path: str) -> str:
        """使用 pylint 检查代码语法规范"""
        from subprocess import run, PIPE
        result = run(
            ["pylint", file_path],
            stdout=PIPE,
            stderr=PIPE,
            text=True
        )
        return f"Pylint 结果:\nstdout: {result.stdout}\nstderr: {result.stderr}"

    # 自定义代码工具：生成单元测试（基于 pytest）
    def unit_test_generator(file_path: str) -> str:
        """为指定文件生成 pytest 测试用例"""
        from langchain_core.prompts import PromptTemplate
        from langchain.chains import LLMChain

        llm = init_kimi_code_llm()
        prompt = PromptTemplate(
            template="为以下代码生成 pytest 测试用例，确保覆盖核心功能和边界条件：\n{code}",
            input_variables=["code"]
        )
        chain = LLMChain(llm=llm, prompt=prompt)

        with open(file_path, "r") as f:
            code = f.read()
        return chain.run(code)

    # 封装自定义工具为 DeepAgents 可识别格式
    custom_tools = [
        Tool(
            name="code_lint",
            func=code_lint_tool,
            description="使用 pylint 检查 Python 代码语法规范，参数为文件路径"
        ),
        Tool(
            name="generate_test",
            func=unit_test_generator,
            description="为指定代码文件生成 pytest 测试用例，参数为文件路径"
        )
    ]

    return base_tools + custom_tools

# --------------------------
# 3. 配置子代理（代码细分任务）
# --------------------------
def get_code_subagents(llm: KimiChat) -> List[Dict[str, Any]]:
    """定义代码任务子代理（测试生成、重构等）"""
    # 测试生成子代理
    test_subagent = {
        "name": "test-generator",
        "description": "生成单元测试代码，确保覆盖率 > 80%",
        "system_prompt": (
            "你是专业测试工程师，使用 pytest 为 Python 代码生成测试用例。"
            "需包含正常场景、边界条件和异常处理，测试代码需可直接运行。"
        ),
        "tools": ["read_file", "write_file", "execute"],  # 依赖的工具
        "model": llm
    }

    # 代码重构子代理
    refactor_subagent = {
        "name": "code-refactor",
        "description": "优化代码结构，提升可读性和性能",
        "system_prompt": (
            "遵循 SOLID 原则重构代码，保留功能不变。"
            "需添加类型注解和文档字符串，移除冗余逻辑，拆分长函数。"
        ),
        "tools": ["read_file", "edit_file", "code_lint"],  # 依赖的工具
        "model": llm
    }

    return [test_subagent, refactor_subagent]

# --------------------------
# 4. 初始化 Code Agent 初始化
# --------------------------
def create_code_agent() -> Any:
    """创建完整的 Code Agent 实例"""
    # 初始化模型
    llm = init_kimi_code_llm()

    # 初始化工具
    tools = get_code_tools()

    # 初始化子代理
    subagents = get_code_subagents(llm)

    # 配置持久化存储（保存项目上下文）
    memory_store = InMemoryStore()  # 生产环境可替换为 Redis
    backend = CompositeBackend(
        default=ModalSandboxBackend(),  # 沙箱执行环境
        routes={"/code_context/": StoreBackend(store=memory_store)}  # 持久化代码上下文
    )

    # 配置上下文压缩中间件（防止 token 溢出）
    middleware = [
        SummarizationMiddleware(
            trigger=("fraction", 0.85),  # 上下文达到 85% 容量时压缩
            keep=("fraction", 0.1)  # 保留 10% 核心内容
        )
    ]

    # 创建 Agent
    return create_deep_agent(
        model=llm,
        tools=tools,
        subagents=subagents,
        system_prompt=(
            "你是专业 Code Agent，擅长 Python/Java 代码开发。"
            "遵循 PEP8 规范（Python），生成可运行代码并自动测试。"
            "复杂任务需先分解为子步骤，必要时调用子代理协作。"
        ),
        backend=backend,
        middleware=middleware,
        interrupt_on={"execute": True, "edit_file": True}  # 危险操作需用户确认
    )
```

## 二、工具扩展：代码安全与质量工具

```python
import subprocess
from deepagents.tools import Tool

# --------------------------
# 1. 依赖管理工具
# --------------------------
def dependency_checker(file_path: str) -> str:
    """检查代码中的未声明依赖并生成安装命令"""
    from pipreqs.pipreqs import get_all_imports  # 需安装 pipreqs

    try:
        imports = get_all_imports(file_path)
        if not imports:
            return "未发现依赖"
        return f"需安装依赖：\npip install {' '.join(imports)}"
    except Exception as e:
        return f"依赖检查失败：{str(e)}"

# --------------------------
# 2. Git 版本控制工具
# --------------------------
def git_commit(message: str) -> str:
    """提交当前代码变更到 Git 仓库"""
    result = subprocess.run(
        ["git", "commit", "-am", message],
        stdout=subprocess.PIPE,
        stderr=subprocess.PIPE,
        text=True
    )
    return f"Git 提交结果：\nstdout: {result.stdout}\nstderr: {result.stderr}"

# --------------------------
# 3. 封装为 DeepAgents 工具
# --------------------------
def get_advanced_code_tools() -> List[Tool]:
    return [
        Tool(
            name="check_dependencies",
            func=dependency_checker,
            description="检查代码中的 Python 依赖并生成安装命令，参数为文件路径"
        ),
        Tool(
            name="git_commit",
            func=git_commit,
            description="提交代码变更到 Git，参数为提交信息"
        )
    ]
```

## 三、执行流程：代码任务处理闭环

```python
import asyncio
from deepagents_cli.execution import execute_task
from deepagents_cli.session import SessionState

# --------------------------
# 1. 任务执行函数
# --------------------------
async def run_code_task(agent: Any, user_input: str) -> None:
    """执行代码相关任务，处理从需求到测试的全流程"""
    # 初始化会话状态
    session_state = SessionState(
        thread_id=str(uuid.uuid4()),
        auto_approve=False  # 关闭自动批准，危险操作需手动确认
    )

    # 调用 DeepAgents 执行流程
    await execute_task(
        user_input=user_input,
        agent=agent,
        assistant_id="code-agent-001",
        session_state=session_state,
        token_tracker=None,
        backend=agent.backend  # 复用 Agent 的沙箱后端
    )

# --------------------------
# 2. 示例：生成 FastAPI 接口并测试
# --------------------------
if __name__ == "__main__":
    # 初始化 Code Agent
    code_agent = create_code_agent()

    # 用户需求：生成带分页的用户列表接口
    user_request = (
        "用 FastAPI 实现一个获取用户列表的接口，包含分页功能（支持 skip 和 limit 参数）。"
        "要求：1. 定义 Pydantic 模型；2. 实现 GET 接口；3. 生成单元测试；4. 检查依赖。"
    )

    # 执行任务
    asyncio.run(run_code_task(code_agent, user_request))
```

## 四、关键配置与安全措施

1. **沙箱隔离配置**  
   通过 `ModalSandboxBackend` 限制代码执行权限，防止恶意操作：

   ```python
   # 自定义沙箱配置（限制资源使用）
   class RestrictedModalSandbox(ModalSandboxBackend):
       def __init__(self):
           super().__init__(
               cpu=1,  # 限制 CPU 核心
               memory="2GB",  # 限制内存
               timeout=30  # 命令超时时间（秒）
           )
   ```

2. **提示词优化（确保代码质量）**  
   在 `system_prompt` 中明确代码规范：

   ```python
   system_prompt = (
       "生成 Python 代码需满足：\n"
       "- 严格遵循 PEP8 规范，变量名使用 snake_case\n"
       "- 所有函数必须包含类型注解和文档字符串\n"
       "- 优先使用 Python 3.10+ 特性（如匹配语句、类型别名）\n"
       "- 避免使用全局变量，通过依赖注入管理资源"
   )
   ```

3. **测试闭环验证**  
   通过子代理实现自动测试与修复：
