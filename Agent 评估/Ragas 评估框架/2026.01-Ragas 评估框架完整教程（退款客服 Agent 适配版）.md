# Ragas 评估框架完整教程（退款客服 Agent 适配版）

Ragas 是面向**RAG/Agent 系统**的量化评估框架，核心通过可解释指标定位检索/生成/事实一致性问题，驱动迭代优化。本教程结合退款客服 Agent 场景，从安装、核心指标、端到端流程、代码示例到优化闭环，帮助快速落地评估体系，精准衡量路由/记忆/安全/执行/评估五层架构表现。

### 一、核心概念与指标体系

Ragas 将评估分为**检索质量**和**生成质量**两大类，适配 Agent 系统需扩展**Agent 特定指标**，以下是退款客服场景的核心指标映射：

| 评估维度            | 核心指标                             | 计算逻辑                       | 退款场景释义                                       | 目标阈值 |
| ------------------- | ------------------------------------ | ------------------------------ | -------------------------------------------------- | -------- |
| **检索/上下文质量** | context_precision（上下文精确率）    | 相关上下文占比                 | 订单信息/用户历史中，与退款请求相关的内容比例      | ≥95%     |
|                     | context_recall（上下文召回率）       | 完整召回相关信息比例           | 是否完整获取订单状态、用户权限、退款历史等关键信息 | ≥98%     |
|                     | context_relevancy（上下文相关性）    | 上下文与问题匹配度             | 检索内容是否解决“这一单”指代、退款条件校验等问题   | ≥90%     |
| **生成/回答质量**   | faithfulness（忠实度）               | 回答与上下文事实一致性         | Agent 输出是否符合订单数据和业务规则，无虚构信息   | ≥95%     |
|                     | answer_relevancy（答案相关性）       | 回答与用户意图匹配度           | 是否直接解决退款请求，无冗余内容                   | ≥90%     |
|                     | factual_correctness（事实正确性）    | 回答与真实情况一致性           | 退款结果、处理流程描述是否符合实际执行结果         | ≥99%     |
| **Agent 特定能力**  | tool_call_accuracy（工具调用准确率） | 正确调用退款 API/风控工具比例  | 执行层是否准确触发 API 调用、重试/降级策略         | ≥98%     |
|                     | agent_goal_accuracy（目标达成率）    | 完成用户退款意图比例           | 从意图识别到退款执行的全流程成功率                 | ≥90%     |
|                     | context_entity_recall（实体召回率）  | 关键实体（订单 ID/金额）覆盖率 | 是否完整提取并使用订单 ID、金额、商品名称等实体    | ≥99%     |

### 三、端到端评估流程（退款 Agent 适配）

#### 1. 评估数据准备（Golden Dataset 构建）

构建包含**用户输入 → 理想输出 → 上下文 → 真实结果**的黄金数据集，覆盖以下场景：

- 正常退款请求（明确订单、符合退款条件）
- 模糊指代退款（“这一单”“昨天买的那个”）
- 闲聊请求（“天气不错”）
- 恶意退款请求（套路话术、虚假信息）
- API 异常场景（退款 API 挂了、超时）

数据格式示例（JSON）：

```json
[
  {
    "user_input": "我想退昨天买的衣服这一单",
    "agent_output": "已为你处理订单ORD123456的退款申请，金额199元将在3个工作日内退回",
    "contexts": [
      "用户ID：U12345，订单列表：[{order_id: ORD123456, product: 衣服, amount: 199, status: 已收货2天, time: 昨天}, {order_id: ORD654321, product: 鞋子, amount: 299, status: 未发货}]"
    ],
    "ground_truth": "成功提交订单ORD123456的退款申请，金额199元，符合7天无理由条件",
    "intent": "refund",
    "risk_level": "safe"
  }
]
```

#### 2. 数据转换与加载

将自定义数据集转换为 Ragas 支持的格式，使用`datasets`库加载：

```python
from datasets import Dataset
import json

# 加载黄金数据集
with open("refund_golden_dataset.json", "r", encoding="utf-8") as f:
    golden_data = json.load(f)

# 转换为Ragas所需格式
ragas_data = {
    "question": [item["user_input"] for item in golden_data],
    "answer": [item["agent_output"] for item in golden_data],
    "contexts": [item["contexts"] for item in golden_data],
    "ground_truth": [item["ground_truth"] for item in golden_data]
}
dataset = Dataset.from_dict(ragas_data)
```

#### 3. 评估执行与结果解析

选择适配退款场景的指标组合，执行评估并解析结果：

```python
from ragas import evaluate
from ragas.metrics import (
    faithfulness, answer_relevancy, context_precision, context_recall,
    factual_correctness, tool_call_accuracy, agent_goal_accuracy
)

# 定义退款场景评估指标集
refund_metrics = [
    faithfulness, answer_relevancy, context_precision, context_recall,
    factual_correctness, agent_goal_accuracy
]

# 执行评估（使用GPT-4作为裁判LLM）
result = evaluate(
    dataset=dataset,
    metrics=refund_metrics,
    llm="gpt-4o"  # 可选："gpt-3.5-turbo"（成本低）或自定义LLM
)

# 转换为DataFrame便于分析
result_df = result.to_pandas()
print("=== 退款Agent评估结果 ===")
print(result_df.describe())  # 输出各指标统计信息
print("\n=== 各样本详细分数 ===")
print(result_df[["question", "faithfulness", "agent_goal_accuracy"]])
```

### 四、退款 Agent 各层专项评估方案

#### 1. 路由层（意图识别）评估

重点：区分退款/闲聊/其他意图的准确率

```python
# 意图识别专项评估指标
from ragas.metrics import answer_relevancy
import numpy as np

# 构建意图识别测试集
intent_test_data = {
    "question": [
        "我想退昨天买的衣服",  # refund
        "今天天气真好",        # chat
        "我的快递到哪了"       # other
    ],
    "answer": [
        "已为你处理退款申请",  # 正确路由
        "是啊，适合外出",      # 正确路由
        "正在查询物流信息"     # 正确路由
    ],
    "contexts": [["意图分类规则：refund=退款关键词，chat=无业务相关，other=物流/查询等"]],
    "ground_truth": ["refund", "chat", "other"]
}

# 评估意图匹配度（自定义指标）
def intent_accuracy_score(answers, ground_truths):
    correct = 0
    for ans, gt in zip(answers, ground_truths):
        if gt.lower() in ans.lower():
            correct += 1
    return correct / len(answers) * 100

# 执行评估
result_df = evaluate(
    Dataset.from_dict(intent_test_data),
    metrics=[answer_relevancy]
).to_pandas()

# 计算意图识别准确率
intent_acc = intent_accuracy_score(
    result_df["answer"].tolist(),
    intent_test_data["ground_truth"]
)
print(f"路由层意图识别准确率：{intent_acc:.2f}%")
```

### 五、评估结果分析与优化闭环

#### 1. 问题定位与根因分析

基于评估结果，聚焦低于阈值的指标，定位问题层：

| 异常指标               | 可能问题层    | 根因示例                    | 优化方向                                  |
| ---------------------- | ------------- | --------------------------- | ----------------------------------------- |
| context_recall 低      | 记忆层        | 未完整获取用户历史订单      | 优化指代解析 Prompt，扩展订单信息检索范围 |
| agent_goal_accuracy 低 | 路由层+执行层 | 意图识别错误或 API 调用失败 | 增强意图分类样本，优化重试/降级策略       |
| faithfulness 低        | 安全层+执行层 | 输出与订单状态不符          | 强化业务规则校验，增加输出事实一致性检查  |

#### 2. 自动化评估流水线构建

```python
# 构建每日评估脚本（示例）
import schedule
import time
from ragas import evaluate
from datasets import load_dataset

def daily_refund_agent_evaluation():
    """每日自动评估退款Agent表现"""
    # 加载最新黄金数据集
    dataset = load_dataset("json", data_files="refund_golden_dataset.json")["train"]

    # 执行评估
    result = evaluate(
        dataset,
        metrics=[
            "faithfulness", "context_recall",
            "agent_goal_accuracy", "tool_call_accuracy"
        ]
    )

    # 输出评估报告
    print(f"=== {time.strftime('%Y-%m-%d')} 退款Agent评估报告 ===")
    print(f"意图识别准确率：{result['agent_goal_accuracy'].mean()*100:.2f}%")
    print(f"上下文召回率：{result['context_recall'].mean()*100:.2f}%")
    print(f"工具调用准确率：{result['tool_call_accuracy'].mean()*100:.2f}%")

    # 阈值告警（示例）
    if result["faithfulness"].mean() < 0.95:
        print("⚠️ 警告：忠实度低于95%，请检查输出事实一致性！")

# 每日凌晨2点执行评估
schedule.every().day.at("02:00").do(daily_refund_agent_evaluation)

# 持续运行
while True:
    schedule.run_pending()
    time.sleep(60)
```

#### 3. 与监控系统集成（Prometheus+Grafana）

将评估指标暴露为 Prometheus 指标，实现实时监控与告警：

```python
from prometheus_client import Gauge, start_http_server
import random
import time

# 定义Prometheus指标
ragas_metrics = {
    "intent_accuracy": Gauge("refund_agent_intent_accuracy", "意图识别准确率"),
    "context_recall": Gauge("refund_agent_context_recall", "上下文召回率"),
    "agent_goal_accuracy": Gauge("refund_agent_goal_accuracy", "目标达成率")
}

def update_metrics_from_ragas(result):
    """从Ragas评估结果更新Prometheus指标"""
    ragas_metrics["intent_accuracy"].set(result["agent_goal_accuracy"].mean())
    ragas_metrics["context_recall"].set(result["context_recall"].mean())
    ragas_metrics["agent_goal_accuracy"].set(result["agent_goal_accuracy"].mean())

# 启动HTTP服务（暴露指标）
start_http_server(8000)

# 模拟评估结果更新
while True:
    # 实际场景：替换为真实Ragas评估结果
    mock_result = {
        "agent_goal_accuracy": [0.98, 0.96, 0.99],
        "context_recall": [0.97, 0.99, 0.98],
        "agent_goal_accuracy": [0.92, 0.95, 0.90]
    }
    update_metrics_from_ragas(mock_result)
    time.sleep(300)  # 每5分钟更新一次
```

### 六、落地优化建议

1. **快速原型阶段**：优先评估核心指标（agent_goal_accuracy+context_recall），验证退款流程可行性
2. **迭代优化阶段**：
   - 第一阶段：完善黄金数据集，覆盖 80%常见场景
   - 第二阶段：接入 CI/CD，每次代码提交自动运行基础评估
   - 第三阶段：构建实时监控，设置指标阈值告警（如 agent_goal_accuracy<90%触发告警）
3. **成本优化**：
   - 日常评估使用 GPT-3.5-turbo（成本低）
   - 周/月全量评估使用 GPT-4o（精度高）
   - 缓存评估结果，避免重复计算
4. **人工兜底**：评估结果异常时，触发人工复核流程，补充黄金数据集样本
